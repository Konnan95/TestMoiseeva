Task 2
1. User Story 

**Как** продавец (пользователь Личного кабинета),  
**я хочу** опубликовать свой товар на витрине маркетплейса,  
**чтобы** покупатели могли увидеть мой товар и оформить заказ.



 2. Use Case

 Публикация товара на витрине маркетплейса

| Элемент | Описание |
| **ID** | UC-01 |
| **Название** | Опубликовать товар |
| **Актор** | Продавец (зарегистрированный пользователь ЛК) |
| **Предусловие** | Продавец авторизован в ЛК; товар создан и заполнен (название, цена, описание, фото, категория, количество) |
| **Постусловие** | Товар становится виден на витрине; продавец получает уведомление об успехе |
| **Основной сценарий** | 1. Продавец заходит в раздел "Мои товары"<br>2. Выбирает нужный товар<br>3. Нажимает кнопку "Опубликовать"<br>4. Система проверяет заполненность всех обязательных полей<br>5. Система проверяет, что товар не дублируется на витрине<br>6. Система публикует товар<br>7. Статус товара меняется на "Опубликован"<br>8. Продавец получает уведомление: "Товар успешно опубликован" |
| **Альтернативный сценарий 1** | Не заполнены обязательные поля → система показывает ошибку: "Заполните все обязательные поля" |
| **Альтернативный сценарий 2** | Товар уже опубликован → система показывает ошибку: "Товар уже находится на витрине" |
| **Альтернативный сценарий 3** | Аккаунт продавца заблокирован → система показывает ошибку: "Доступ запрещен. Обратитесь в поддержку" |
| **Исключение** | Сервис временно недоступен → система показывает: "Сервис временно недоступен. Попробуйте позже" |
